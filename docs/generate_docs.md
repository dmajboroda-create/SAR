# Генерація документації для проєкту SAR

## Інструмент: Doxygen

Для проєкту SAR (C++) обрано **Doxygen** — стандартний інструмент генерації
документації для C/C++ проєктів.

## Встановлення

### Windows (через winget)
```bash
winget install doxygen.doxygen
```

### Перевірка встановлення
```bash
doxygen --version
```

## Генерація документації

У корені репозиторію виконайте:

```bash
doxygen Doxyfile
```

Документація буде згенерована у папку `docs/code/html/`.

Відкрийте `docs/code/html/index.html` у браузері для перегляду.

## Стандарт документування

Проєкт використовує стиль Doxygen з командами `@`:

```cpp
/**
 * @brief Короткий опис функції (одним реченням).
 *
 * Детальний опис: що робить функція, як працює алгоритм,
 * які формули використовує.
 *
 * @param t   Момент часу (секунди)
 * @param y   Вектор стану системи
 * @return    Вектор похідних
 *
 * @throws std::runtime_error якщо параметр T == 0
 *
 * @note    Важлива примітка про особливості використання.
 * @warning Попередження про небезпечне використання.
 *
 * @see RKSolver::step()
 */
```

## Правила документування для нових розробників

1. **Кожен клас** повинен мати `@brief` та детальний опис з поясненням
   математичної моделі або архітектурного рішення.

2. **Кожен публічний метод** повинен мати:
    - `@brief` — одне речення опису
    - `@param` — для кожного параметра з одиницями виміру
    - `@return` — опис значення, що повертається
    - `@throws` — якщо функція може кинути виняток

3. **Математичні формули** записуються у форматі LaTeX між `@f[` і `@f]`:
   ```
   @f[ F(t) = F_0 \cdot e^{-\alpha t} @f]
   ```

4. **Групування** пов'язаних членів класу через `@defgroup` та `@{` / `@}`.

5. **Перехресні посилання** між пов'язаними класами через `@see`.
